Архитектура

Один интерфейс для сети магазинов, локальная обработка видео и AI рядом с камерами

kVision разделяет систему на понятные контуры: оператор видит единый интерфейс, CentralServer отвечает за доступ и маршрутизацию, локальный архив и камеры остаются на стороне магазина, а Neuro работает как внутренний CV-сервис. Маршрут остается простым и управляемым: Frontend -> CentralServer -> TobaccoServer -> Neuro.

01 / Пользователь

Оператор сети

Работает в одном интерфейсе: выбирает магазин, открывает стримы, проверяет архив, видит нарушения и статистику без перехода между локальными серверами.

  • единая точка работы для контроля качества;
  • быстрый переход к нужной торговой точке;
  • понятная доказательная база по каждому событию.
KV оператор
02 / Frontend

Рабочее место

Frontend показывает продукт как цельный сервис, но не ходит напрямую в локальные серверы магазинов. Все запросы проходят через центральный контур, поэтому доступы и маршруты остаются управляемыми.

  • стримы, архивы, фиксации и отчеты в одном интерфейсе;
  • меньше ручных действий для оператора;
  • единый UX для всех торговых точек.
03 / CentralServer

Единая точка входа

CentralServer держит пользователей, роли, аудит и список магазинов. Он принимает запрос от интерфейса, определяет нужную торговую точку и проксирует его дальше в правильный локальный контур.

  • централизованный контроль доступа;
  • единый формат данных для сети;
  • масштабирование без усложнения рабочего места.

04 / TobaccoServer

Локальный контур магазина

TobaccoServer находится рядом с камерами: управляет RTSP-потоками, HLS-стримингом, видеоархивом, зонами контроля, заданиями аналитики и файлами фиксаций конкретной точки.

  • видео не гоняется через центральный сервер без необходимости;
  • архив и доказательная база остаются локально;
  • точка продолжает быть понятной частью общей сети.

05 / Neuro

AI-анализ

Neuro работает как внутренний CV-сервис: получает кадры от локального сервера, анализирует их моделями компьютерного зрения и возвращает результат, который затем превращается в фиксацию.

  • детекция объектов, людей и действий;
  • классификация событий и помощь с false positive;
  • AI встроен в процесс, а не вынесен в отдельный интерфейс.

06 / Полная схема

Frontend -> CentralServer -> TobaccoServer -> Neuro

Пользователь не работает с локальными серверами напрямую. Центральный контур управляет доступом и маршрутом, локальный сервер магазина держит камеры и архив, Neuro возвращает результат анализа обратно в цепочку фиксаций.

Схема работы

Ежедневный контроль без ручного просмотра часов видео

kVision помогает не только смотреть камеры, но и выстраивать процесс: оператор закрывает фиксации, администратор управляет доступами и точками, а руководитель видит повторяющиеся проблемы и принимает решения по сети.

01

Оператор закрывает фиксации

Проверяет поток или архив, открывает карточку события, смотрит подтверждающий фрагмент и передает инцидент дальше по внутреннему регламенту.

02

Администратор управляет магазинами

Настраивает пользователей, роли, точки контроля, камеры и доступы так, чтобы каждая команда видела только свои магазины и рабочие разделы.

03

Руководитель видит повторяющиеся проблемы

Сравнивает магазины, отслеживает динамику нарушений, видит проблемные зоны и понимает, где нужен инструктаж, проверка или изменение процесса.

04

Сеть улучшает процессы

Данные из фиксаций превращаются в регулярный цикл управления: обнаружили событие, подтвердили, нашли причину, проверили результат после изменений.
